# gosplan-api-test

Тестовый проект для оценки **ГосПлан API v2** (`v2.gosplan.info`) как источника данных по госзакупкам (44-ФЗ / 223-ФЗ).

## Что внутри

- `index.html` — карточка закупки (поиск по номеру, отображение НМЦК, ОИК, аванса, этапа)
- `api/proxy.js` — серверный прокси для Vercel (API не отдаёт CORS-заголовки)

## Выводы по ГосПлан API

**Плюсы:**
- Бесплатный доступ до 01.07.2026 (600 req/min)
- REST/JSON, простая интеграция
- Swagger-документация: https://swagger.gosplan.info
- Покрытие: 44-ФЗ, 223-ФЗ, ПП РФ 615

**Проблемы (критичные):**
- **Поле `stage` ненадёжно** — закупка завершена год назад, а API отдаёт `stage: 1` (подача заявок). Пример: `32514883824` (223-ФЗ)
- **Query-поиск для 223-ФЗ не работает** — `?purchase_number=` возвращает пустой массив, хотя path-эндпоинт `/purchases/{num}` находит закупку
- **CORS отсутствует** — для браузерного приложения нужен свой прокси-сервер
- Данные загружаются с задержкой до 2 часов после публикации в ЕИС

## Альтернативы

| Источник | Формат | Бесплатно | Примечание |
|----------|--------|-----------|------------|
| **ГосПлан** | REST/JSON | до 01.07.2026 | stage ненадёжный, query 223-ФЗ не работает |
| **DaMIA** | REST/JSON | 100 запросов | протоколы бывают некорректны |
| **NewDB** | REST/JSON | 100 запросов | только контракты |
| **zakupki.gov.ru** | SOAP/XML | бесплатно | первоисточник, сложная интеграция |
